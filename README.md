# Actividad de GitHub Classroom: Micropráctica ARM64

## Rol del curso
Esta actividad está diseñada para estudiantes de arquitectura de computadoras y sistemas embebidos en nivel licenciatura/TSU.

## Propósito general
Implementar una micropráctica en **ARM64 Assembly** enfocada en manipulación de registros, macros y control de flujo, con integración opcional ligera en **Bash** para automatizar compilación y ejecución.

## Entregables de la actividad (equipo o individual)
1. Código fuente ARM64 (`src/main.s`) con al menos **1 macro**.
2. Script opcional de ejecución (`run.sh`) sin dependencias pesadas.
3. Evidencias de pruebas según `docs/plan_de_pruebas.md`.
4. Reflexión crítica sobre uso de IA en `docs/reflexion_ia.md`.

## Alcance técnico (micropráctica)
- Longitud sugerida de implementación: **≤150 líneas**.
- Funcionalidades esperadas: **3 a 5**.
- Sin Python, sin Docker, sin frameworks pesados.

## Flujo sugerido en GitHub Classroom
1. Aceptar asignación y clonar repositorio.
2. Crear rama `feature/micropractica-arm64`.
3. Implementar solución mínima funcional.
4. Ejecutar plan de pruebas.
5. Abrir Pull Request con evidencias.
6. Recibir retroalimentación y ajustar.

## Criterios de evaluación rápidos
- Correcto uso de instrucciones ARM64 y registros.
- Presencia y utilidad de al menos una macro.
- Claridad del repositorio y documentación.
- Evidencias de pruebas reproducibles.
- Reflexión ética/técnica del uso de IA.

Para detalles completos revisa la carpeta `docs/`.
